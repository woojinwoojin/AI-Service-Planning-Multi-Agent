# AI 서비스 기획 보조 **Multi-Agent**

한 줄 아이디어를 입력하면, 여러 AI Agent가
**웹 근거를 수집하고 검토해 서비스 기획서로 완성**합니다

<br>

가천대학교 최우진
현장 미러형 프로젝트 · **1차 발표** · 2026-07-21
FastAPI + LangGraph

> 💬 반복할 핵심 문장: "Multi-Agent라서 자동으로 더 뛰어난 게 아니라, **역할과 중간 산출물을 분리**해 조사·작성·검토 과정을 **추적하고 통제**할 수 있게 만들었습니다. 현재 실험에서는 전체 파이프라인의 효과를 확인했고, 다음 실험에서는 동일 검색자료로 **역할 분리 자체의 효과**를 검증합니다."

---

## 1. 문제

```
시장·고객·경쟁·리스크를 혼자 조사하기 어려움
                  ↓
단일 LLM에 한 번에 요청
                  ↓
그럴듯하지만 출처·검증이 부족한 초안
```

특히 **근거 없는 통계를 지어냄**(환각):

## “2022년 조사에 따르면 60%가… 70%는…”

> 💬 "핵심 통증은 '그럴듯함'이 아니라 '믿을 수 있느냐'입니다."

## 2. 제안 서비스

아이디어 한 줄 → **14개 항목으로 구성된 서비스 기획서** (웹 검색 출처 인용)

| 근거 수집 | 품질 개선 | 서비스 기능 |
|---|---|---|
| Tavily 웹 검색 | Reviewer 100점 평가 | 사용자 직접 수정 |
| 출처 URL 인용 | 자동 재작성·최종 재평가 | 실행 이력 저장 |
| | 근거 일치성 검토 | MD·JSON·Word·**PPT** 내보내기 |
| | | 비용·토큰·지연 관측 |

> 💬 세 덩어리로: 근거를 모으고 → 품질을 검토하고 → 쓸 수 있게 내보냅니다.

## 4. 역할 특화 Multi-Agent 워크플로

> 동일한 기반 LLM을 사용하되, 각 호출을 **서로 다른 목표·도구·입출력 스키마**를 가진 역할 특화 Agent로 분리한 워크플로

**입력·조사** → preprocess → Research(웹검색·출처 수집)

**분석**(독립 분기 **병렬 실행**) → Competitor · Customer · PESTEL · SWOT · Business Model · Risk

**작성·개선** → Draft → Reviewer → (총점<90 시 재작성) → Polish → Final Reviewer

**검증·출력** → Verify(주장↔근거 대조) → 최종 기획서

<br>

- 오케스트레이션: LangGraph `StateGraph`
- 각 Agent는 **사용자 입력과 앞 단계에서 수집된 자료**를 기반으로 역할별 결과 생성
- **개별 Agent 실패가 전체 파이프라인 중단으로 바로 이어지지 않도록** fallback 설계

> 💬 "기반 모델은 같지만 Research는 검색 도구·조사 스키마를, Reviewer는 평가표·수정 판단 권한을 갖도록 소프트웨어 역할을 분리했습니다."

---

## 5. 비교 실험 — 검색 없는 단일 프롬프트 vs 근거 기반 Multi-Agent

**평가 비교 조건 통제:**

- 동일 입력 · 동일 출력 서식 · 동일 평가표 · 동일 심판 모델(gpt-4o-mini)
- **6개 주제 · 결과당 3회 평가 평균**
- 단, 단일 방식과 Multi-Agent 방식의 **검색·호출·검토 구조는 서로 다름**

> 💬 "평가 쪽 조건은 고정했고, 두 방식의 처리 구조 차이(검색·검토)는 그대로 둔 비교입니다."

---

## 6. 현재 비교에서 가장 명확한 차이 — 고유 출처 URL

<br>

### 단일 프롬프트 **0개**  →  Multi-Agent **평균 5개**

<small>단일 방식은 검색 도구 없음 / Multi-Agent 방식은 Tavily 검색 포함</small>

<br>

- LLM 평가 총점: 84.9 → **86.5 (+1.6)**
- 6개 주제 중 **5개**에서 Multi-Agent 우위

> 💬 헤드라인은 출처 0 → 평균 5. 점수는 보조 지표. 이 실험이 우선 보여주는 것은 "웹 검색·출처 삽입이 포함된 파이프라인이 검색 없는 단일 프롬프트보다 추적 가능한 근거를 더 많이 포함한다"는 점.

## 7. 생성 모델을 바꿔도 출처 차이는 유지

별도의 **3주제** 실험 — 심판 고정, 생성 모델만 변경:

| 생성 모델 | 점수 차이 | 출처(단일 / Multi) |
|---|---|---|
| gpt-4o-mini | +1.8 | 0 / 평균 5 |
| gpt-4o | +1.6 | 0 / 평균 5 |

- 두 모델 조건 모두 **출처 0 vs 평균 5 유지**
- 점수 차이는 +1.6~+1.8로 유사
- ⚠️ 표본이 작으므로 **일반화가 아닌 방향성 확인 결과**로 해석

---

## 8. 정직한 범위 (과장하지 않기)

- 본 비교는 **Agent 분업 효과와 웹 검색 효과를 함께** 측정한다
  (분업 그 자체의 효과만 분리한 것은 아님)
- 출처 URL 수는 **포함 개수**이며, 실제 내용의 사실성을 보장하지 않는다
- 다중 모델 결과는 **3주제 기반의 방향성 확인** 결과다

> 💬 자세한 내용은 부록/ Q&A에서. 본문에선 이 세 줄만.

## 9. 안정성 · 환각 대응

**안정성** — 부분 실패에도 완주하도록 설계
- LLM 호출 실패(레이트리밋/네트워크) 재시도 후 fallback으로 흡수, 로그에 정직 표기
- 미완성 섹션은 `[더미]` 대신 **원인별 안내**("AI 서버 혼잡 — 다시 생성")

**환각 대응 (2겹)**
1. 생성 억제 — 근거 없는 수치·"조사에 따르면" 생성 억제 규칙(프롬프트)
2. 탐지·표시 — 수집 근거에서 확인되지 않은 주장을 **주황색 '근거 미확인'**으로 표시
   (⚠️ '거짓'이 아니라 '수집한 검색 근거에서 확인되지 않음' — 검증 범위를 정직하게 명시)

> 💬 테스트 사례에서 근거 없는 "2022년 조사·60%·70%" 문구 제거를 확인(단일 사례).
> 검증 판정은 '수집 검색 요약과의 일치'이며 URL 원문 사실 검증이 아님을 UI·문서·JSON에 동일하게 표시.

## 12. 한계 · 향후 계획 (핵심 3)

1. **동일 검색자료 기준선 추가**
   → 웹 검색 효과와 Agent 분업 효과를 **분리** 측정
2. **문서 재생성 병목 단축**(병렬화는 완료 −16%)
   → 전체 재작성 대신 **섹션 단위 수정**(남은 최대 병목)
3. **근거 구조화·출처 품질 검증 강화**
   → Evidence Registry(착수) → 주장별 근거 연결 → URL 원문 대조

> 💬 병렬화는 이미 완료(−16%). 다음 병목은 문서 재생성. 유료화 등은 사업화 단계 과제(부록).

## 부록 E — 예상 질문 대비

**Q. 같은 LLM에 다른 프롬프트를 여러 번 준 것인데, 왜 Multi-Agent인가요?** ⭐
기반 LLM은 같지만 각 단계에 **독립적인 목표·사용 가능한 도구·입출력 스키마·상태**를 부여했습니다. 서로 다른 AI가 협상하는 자율형 Multi-Agent라기보다, 동일 LLM을 **역할별 소프트웨어 구성요소로 분리한 역할 특화 Multi-Agent 워크플로**에 가깝습니다. 목적도 "여러 AI가 있다"가 아니라, 작업을 **단계별로 통제하고 중간 결과를 추적·검증**할 수 있게 만드는 데 있습니다.

**Q. 검색을 안 준 단일 LLM과 비교하면 당연히 출처가 없지 않나요?**
맞습니다. 현재 실험은 검색·단계별 검토를 포함한 **전체 파이프라인의 효과**를 우선 확인한 것입니다. 다음 실험에서 단일 LLM에도 **동일 검색자료를 제공**해 검색 효과와 분업 효과를 분리하겠습니다.

**Q. 점수 차 +1.6점이면 의미가 작지 않나요?**
점수는 보조 지표입니다. 더 명확한 차이는 출처 URL 0 → 평균 5입니다. 다만 URL 수가 품질을 보장하진 않아, 접근성·관련성을 별도 평가할 예정입니다.

**Q. 실행 시간이 느리지 않나요?**
독립 분석 Agent를 **병렬화해 이미 −16%(127.4→106.8s) 단축**했고, 분석 구간은 −41%(35.7→21.0s)입니다. 남은 최대 병목은 문서 재생성(재작성+편집)이라, 다음으로 **섹션 단위 수정**을 적용해 추가 단축할 계획입니다.

**Q. LLM이 만든 문서를 LLM이 평가하면 믿을 수 있나요?**
