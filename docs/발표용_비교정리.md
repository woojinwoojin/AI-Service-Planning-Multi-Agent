## 결론 한 줄

**Multi-Agent의 가치는 "점수"가 아니라 "추적 가능한 근거"에 있다.** 단일 LLM은 검색을 하지 않아 출처 URL 0건, Multi-Agent는 실제 출처 URL 5건을 인용한다 — 이 격차는 모델을 바꿔도 뒤집히지 않는 **구조적** 차이다.

> 지표 정의(정직성): 이 "출처 URL 수"는 기획서 본문의 **고유 URL 포함 개수**를 하드 카운트한 값이다. URL의 실제 접속 가능성·내용 사실성까지 검증한 값은 아니다(그건 별도 과제). 그럼에도 단일 LLM은 URL 자체가 0이라는 점에서 대비는 유효하다.

---

## 발표 프레이밍 (헤드라인 → 보조 순서)

1. **헤드라인 = 객관 지표: 기획서에 포함된 고유 출처 URL 수 `0 vs 5`**
   - 성질: 객관적·구조적·이분법적. 단일 LLM은 검색 자체를 안 하므로 구조적으로 0.
   - 표본·모델을 바꿔도 **절대 뒤집히지 않음** → 가장 방어력 높은 근거.

2. **보조 = LLM 심판 점수: 단일 84.9 vs Multi 86.5 (+1.6)**
   - 정직한 한계 명시: 작은 LLM 심판은 "유창하지만 근거 없는 글"과 "실제 인용된 글"을 잘 구분하지 못해 격차가 작게 나온다.
   - **6주제 중 5개에서 Multi 우위**(나머지 1개 주제는 -0.3점 근소 열세 — 과장 없이 정직하게).

3. **모델 등급 스토리 (다중모델 표)**
   - gpt-4o-mini: 점수 격차 **+1.8**, gpt-4o: **+1.6** → 모델이 강해지면 점수 격차는 **줄어든다**.
   - 그러나 **출처 격차는 그대로 0 vs 5**.
   - 메시지: *"강한 모델도 스스로 검색하지는 않는다."* 점수가 줄어드는 것이 오히려 Multi-Agent의 구조적 가치를 증명한다.

## 알려진 약점과 대응

- **다중모델 표가 3주제뿐** → 발표에서 "모델 등급 무관 우위 유지의 **방향성 확인용**"이라고 정직하게 라벨링.
- **측정 범위(중요)**: 이 비교는 "단일 프롬프트 1회" vs "웹검색+단계별 분석·검토를 포함한 **파이프라인 전체**"다. 즉 *분업 효과*와 *웹검색 효과*가 섞여 있다. 발표에서는 **"단일 프롬프트 방식 vs 근거 기반 Multi-Agent 파이프라인"**으로 명명하고, "역할 분리 그 자체의 효과"라고 과장하지 않는다. 순수 분업 효과 분리는 `단일 Agent + 동일 웹검색 자료` 기준선 추가로 향후 가능.
- 청중이 표본을 강하게 문제 삼거나 발표 중 오점이 드러나면 → **차선책 B 발동**.

## 차선책 B (보류) — 발동 조건과 방법

- **발동 조건**: 3주제 표본이 발표 방어에 부족하다고 판단될 때 (오점·질의 예상).
- **방법**: `run_multimodel.py`의 `TOPICS`를 3 → 6개로 확장(메인 비교의 6주제와 동일하게) 후 재실행.
  - 심판 고정(gpt-4o-mini) 유지 → 행 간 공정 비교 보존.
  - 이어하기 지원((모델,주제)별 즉시 저장)이라 중단돼도 진행분 보존.
- **비용 감안**: gpt-4o 생성 포함이라 크레딧 소모 有(잔여 ~$11 내에서 충분). 시간 ~20분+.
- **주의**: gpt-4o는 점수 격차가 더 **작아질** 수 있음(단일도 잘 씀). 헤드라인은 여전히 출처 0 vs 5로 유지할 것.
