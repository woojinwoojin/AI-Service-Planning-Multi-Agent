# 진척도 스냅샷 — 2026-07-24

> 기준: 개발 로드맵 v2(Phase 0~8). 아래 수치는 **정성 평가(추정치)**이며, 기능 개수가 아니라
> "실제 서비스 출시까지 필요한 작업량"을 기준으로 가중한 판단이다.

## 전체 진척도

| 기준 | 진척도 |
|---|---|
| **전체 로드맵(출시까지 포함)** | **약 62%** (범위 60~65%) |
| 핵심 백엔드·Multi-Agent 구조 | 약 80% |
| 평가·신뢰도 체계 | 약 75% |

기준을 나누면 차이가 크다. 기능 수만 세면 70% 이상처럼 보이지만, 남은 작업(UI·운영·배포·통합 테스트)이 서비스화에 필요한 무거운 작업이라 전체 평가는 약 62%가 현실적이다.

## 단계별 평가

| 단계 | 진척도 | 판단 |
|---|---|---|
| Phase 0 기준선 고정 | 100% | baseline-v1, 재현 명령, 실험 서명 완료 |
| Phase 1 평가 기준선 | 90% | 평가 세트·루브릭·변동성 측정 완료. 사람 평가 보정 미완료 |
| Phase 2 Multi-Agent 구조 개혁 | 65% | Evidence Registry·구조화 이슈·섹션 수정 완료. Artifact Contract·선택적 Agent 재실행 남음 |
| Phase 3 정보 신뢰도 | 55% | Tier 2·GT 스모크 평가 완료. URL 원문 검증(Tier 3) 미착수 |
| Phase 4 품질 게이트 | 85% | release gate·최고 버전 채택·Reviewer 모델 분리 완료. 사람 기준 임계값 보정 남음 |
| Phase 5 API·State 계약 | 70% | State 버전·과거 데이터 호환·Architecture 문서 완료. 오류·Streaming·OpenAPI 계약 남음 |

| Phase 7 E2E·보안·부하·사용자 테스트 | 10~15% | 단위·회귀 테스트는 강하나 E2E·부하·사용자 테스트 대부분 남음 |
| Phase 8 배포 | 0~5% | Staging·Production CD·rollback 미착수 |

## 현재 위치

단순 MVP 단계를 이미 넘었다. **연구·개발용 백엔드 프로토타입은 상당히 완성**됐지만, 실제 사용자가 안정적으로 쓰는 서비스로 만들기 위한 **UI·운영·배포 단계가 남은 상태**다.

처음 계획한 핵심 기능 기준으로는 거의 완성:
- 여러 Agent의 분석 · 실제 검색 출처 · 병렬 실행
- Reviewer와 재작성 · 섹션 단위 수정 · Evidence Registry · Tier 2 검증
- 품질 게이트 · 이력 및 문서 출력 · 성능·품질 비교 실험

전체 로드맵 후반부에 남은 무거운 작업:
- 공통 Artifact 계약 · 문제 유형별 담당 Agent 선택적 재실행
- URL 원문 추출과 최신성·출처 충돌 검증(Tier 3)
- API 오류 및 SSE 이벤트 계약 · UI 전면 개선
- E2E·부하·보안 테스트 · 사용자 테스트 · Staging/Production 배포

## 오늘(07-24) 작업으로 오른 부분

오늘 작업 전에는 전체 기준 약 45~50%로 추정. 아래가 한꺼번에 진행되며 **약 12~15%p 전진**한 것으로 본다.

- 섹션 단위 재작성 완성 및 실측 · 조건부 Polish
- Tier 2 평가 · 품질 게이트 · 최고 버전 유지
- Writer·Reviewer 모델 분리 · State 버전과 이전 기록 호환 · Architecture 문서 최신화

특히 **Phase 4를 거의 끝내고 Phase 5의 핵심 기반까지** 들어간 점이 크다.

## 앞으로의 우선순위 (제출 완성도 기준)

잔여 항목을 같은 비중으로 처리할 필요는 없다. 제출 프로젝트 완성도 기준 아래 네 묶음이 중요:

1. **Phase 5 마무리** — SSE 이벤트·오류 응답·OpenAPI 계약
2. **UI 정리** — 핵심 실행·결과·근거·수정 흐름 개선
3. **E2E 테스트** — 입력부터 다운로드까지 자동 검증
4. **Staging 배포** — 외부에서 실제 실행 가능한 형태 확보

Artifact Contract·동적 Agent 재실행·Tier 3은 기술적 가치는 높지만 **제출 전 필수라기보다 후속 고도화 항목**으로 남겨도 충분하다.

## 한 문장 요약

> 전체 로드맵은 약 62%, 핵심 Multi-Agent 백엔드는 약 80% 완료됐으며, 앞으로는 새로운 분석 기능보다 **API 안정화·UI·통합 테스트·배포**에 집중해야 하는 단계다.
