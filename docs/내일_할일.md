## 현재 아키텍처 (12-노드 Multi-Agent)

```text
사용자 입력
  → preprocess (입력 구조화 함수)
  → Research        (Tavily 웹검색 → 시장조사 + 출처)
  → Competitor      (경쟁사 강점/약점/포지셔닝/차별화)
  → Customer        (페르소나·Pain·니즈·JTBD)
  → PESTEL          (6요인 × 4항목 표)
  → SWOT            (강점/약점/기회/위협)
  → Business Model  (수익원/가격/비용/핵심지표)
  → Risk            (유형별 리스크 + 가능성/영향/대응)
  → Draft Writer    (고정 14섹션 + 출처 인용)
  → Reviewer        (5항목 100점 + 개선지시)
  → (총점<90) Draft 재작성 1회
  → Polish          (섹션 중복 제거·연결 보강)
  → Final Reviewer  (최종본 재평가 → 표시 점수, 초안 대비 변화)
  → Verify          (기획서 주장 ↔ 근거 대조, 지지율)
  → 최종 기획서 + Agent별 산출물 + 관측치
```

- **오케스트레이션**: LangGraph, 전 노드 `_safe` 래핑(완주 보장)
- **services**: `llm`(provider/모델·재시도·fallback·관측), `search`(Tavily), `compare`(단일vs멀티), `markdown_export`/`docx_export`, `store`(SQLite 이력), `usage`(관측성)
- **API**: `/`(UI) · `/health` · `/models` · `/run` · `/revise` · `/run/save` · `/export/docx` · `/projects` · `/projects/{id}`
- **산출물**: Markdown · JSON · Word(.docx)
- **품질장치**: 스키마 검증(`_validate`), 실측 비용(~$0.01/실행, ~2분, mini)
- **비교 baseline**: `single_agent`(단일 프롬프트 1회) — 동일 서식으로 공정 비교
- **테스트**: `pytest` 44개 (LLM 호출 없이 검증 로직 커버)
