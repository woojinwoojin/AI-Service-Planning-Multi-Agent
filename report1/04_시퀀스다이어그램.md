# 04. 시퀀스 다이어그램

> 중간 보고서의 「시스템 설계 — 주요 흐름」 절에 사용. Mermaid ` ```sequenceDiagram ` 블록.

---

## 1. 기획서 생성 (핵심 흐름, `/run/stream` · 병렬 모드)

사용자가 실행하면 SSE로 노드 완료가 실시간 전송되고, 완료 시 결과가 이력에 저장된다.

```mermaid
sequenceDiagram
    autonumber
    actor U as 사용자(브라우저)
    participant API as FastAPI<br/>/run/stream
    participant WF as LangGraph<br/>Workflow
    participant LLM as LLM<br/>(OpenAI)
    participant TV as Tavily<br/>웹검색
    participant DB as SQLite<br/>이력

    U->>API: POST /run/stream (아이디어 입력)
    API->>WF: run_workflow_stream(input, mode=parallel)

    WF->>WF: preprocess (입력 구조화)
    API-->>U: SSE {node: preprocess}

    WF->>TV: web_search(시장 쿼리)
    TV-->>WF: 출처 URL·스니펫
    WF->>LLM: Research 프롬프트(+검색결과 격리)
    LLM-->>WF: 시장조사 JSON
    API-->>U: SSE {node: research}

    par 독립 분석 4분기 병렬 실행
        WF->>TV: web_search(경쟁 쿼리)
        WF->>LLM: Competitor → SWOT
    and
        WF->>LLM: Customer 분석
    and
        WF->>LLM: PESTEL → Risk
    and
        WF->>LLM: Business Model
    end
    API-->>U: SSE {node: competitor/customer/...}

    WF->>LLM: Draft (14섹션 작성 + 실제 출처 인용)
    API-->>U: SSE {node: draft}
    WF->>LLM: Reviewer (100점 평가)
    API-->>U: SSE {node: reviewer}

    alt 총점 < 90
        WF->>LLM: Revise (재작성 1회)
    else 총점 >= 90
        WF->>WF: finalize (초안 확정)
    end

    WF->>LLM: Polish (일관성 편집)
    WF->>LLM: Final Reviewer (최종본 재평가)
    WF->>LLM: Verify (주장 ↔ 근거 대조 → 근거 확인율)
    API-->>U: SSE {node: polish/final_reviewer/verify}

    WF-->>API: 최종 State(기획서·근거·점수·관측치)
    API->>DB: save_run(state)
    DB-->>API: project_id
    API-->>U: SSE {type: done, result}
```

> **fallback**: 각 노드는 `_safe()` 로 감싸져 있어 LLM/검색 실패 시 예외를 흡수하고 fallback 값으로 진행 → 파이프라인은 항상 완주하며 실패는 `run_status` 로 표면화된다.

## 2. 입력 자동완성 (`/suggest`)

빈 항목만 채우고 사용자 입력은 보존한다. 비교 모드는 전 항목에 제안을 생성한다.

```mermaid
sequenceDiagram
    autonumber
    actor U as 사용자
    participant API as FastAPI<br/>/suggest
    participant S as suggest.py
    participant LLM as LLM

    U->>API: POST /suggest (project_name, 기존 입력, compare?)
    API->>S: suggest_fields(name, memo, existing, compare)
    S->>LLM: 추천 프롬프트(기존 입력은 문맥으로만)
    LLM-->>S: 항목별 제안 + 이유 + 확신도
    alt compare = false (기본)
        S-->>API: 빈 항목만 채운 제안
    else compare = true
        S-->>API: 전 항목 제안(내 값 vs AI 비교용)
    end
    API-->>U: 제안·이유·확신도
    U->>U: 항목별 유지 / AI 적용 / 합치기 선택
```

## 3. 사용자 수정 (Human-in-the-Loop, `/revise`)

수정 요청을 반영해 재작성하고, `/run` 후반부와 동일하게 재처리·재평가한다.

```mermaid
sequenceDiagram
    autonumber
    actor U as 사용자
    participant API as FastAPI<br/>/revise
    participant DB as SQLite
    participant DW as Draft Writer
    participant RV as Reviewer/Verifier

    U->>API: POST /revise (draft, 수정요청, project_id?)
    opt project_id 있음
        API->>DB: get_project(id) → 근거·출처 유지
    end
    API->>DW: revise(state) 재작성
    API->>DW: polish (일관성 편집)
    API->>RV: final_reviewer 재평가 + verify 근거검증
    API->>API: run_status 재판정
    alt 기존 프로젝트
        API->>DB: update_run(id, state)
    else 신규
        API->>DB: save_run(state)
    end
    API-->>U: 수정된 최종본 + 점수 + 관측치
```
